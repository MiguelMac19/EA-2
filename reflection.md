# EA2 — Reflection

## Activity Description

For this assignment I continued working through TryHackMe's Cyber Security 101
learning path, completing three rooms: Linux Fundamentals Part 1, Cryptography
Basics, and Windows Command Line. Each room combined short reading sections with
hands on tasks in a virtual environment. Linux Fundamentals Part 1 covered
filesystem navigation and file permissions. Cryptography Basics introduced symmetric and
asymmetric encryption, hashing, and Diffie-Hellman key exchange. Windows Command Line
covered CMD and PowerShell for filesystem navigation, process management, and system
queries.

## Technical Decisions

I stayed on TryHackMe since I already had an account from EA-01.
I chose these three rooms because they cover both major operating environments,
which is directly relevant to IT Support where you rarely work on just one OS. I picked
Cryptography Basics as the middle room because it was the most conceptually unfamiliar
of the three. When the asymmetric key exchange material got abstract, I slowed down and
read the explanations fully rather than skipping ahead, which helped when later tasks
built on earlier concepts.

## Contributions

I completed this activity independently. All room selection and pacing decisions were
made on my own.

## Quality Assessment

The Windows Command Line room was more useful than I expected. I've been working almost
entirely in Linux environments this semester through UTM, Docker, and forensics labs, so
I assumed Windows CLI would feel easy. Seeing how CMD and PowerShell handle
the same operations I do in bash made clear how much of my muscle memory won't transfer
directly in a helpdesk role where Windows endpoints dominate.

Cryptography Basics was the most valuable conceptually. Working through key exchange
scenarios made PKI feel less like a black box, which connects directly to Security+
material I'm currently studying. If I could redo this I would have started earlier to
fit in a fourth room and taken better notes during the Windows CLI tasks. Overall these
rooms reinforced that IT Support requires real cross platform fluency, not just
familiarity with one environment.
